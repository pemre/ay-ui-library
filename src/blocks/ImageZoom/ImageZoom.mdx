# ImageZoom

A mouse-tracking zoom-on-hover image component for React. Hover over the image and the area under your cursor magnifies — letting users inspect detail without leaving the page.

<Canvas of={ImageZoomStories.Default} />

## Prerequisites

ImageZoom uses a **hybrid styling approach**: Tailwind CSS utility classes handle layout and the scale transform, while the library's CSS custom property system handles theming. Your consumer project must have Tailwind CSS configured for the hover-zoom effect to work.

```bash
npm install tailwindcss @tailwindcss/vite
```

Make sure the Tailwind classes used by ImageZoom are included in your content paths. If you use Tailwind v4, no extra configuration is needed beyond the standard setup. For Tailwind v3, add the library's source to your `content` array:

```js
// tailwind.config.js (v3)
module.exports = {
  content: [
    "./src/**/*.{ts,tsx}",
    "./node_modules/ay-ui-library/dist/**/*.js",
  ],
};
```

> Without Tailwind configured, the image still renders correctly but the hover-zoom and layout utilities will not apply.

## Usage

```tsx
import { ImageZoom } from "ay-ui-library";
import "ay-ui-library/dist/style.css";

function App() {
  return (
    <ImageZoom
      src="https://example.com/photo.jpg"
      alt="Mountain landscape at sunset"
    />
  );
}
```

### Custom Zoom Level and Transition

```tsx
<ImageZoom
  src="/images/product-detail.jpg"
  alt="Product close-up"
  config={{ zoomLevel: 2, transitionDuration: 500 }}
/>
```

### With Custom Class Names

```tsx
<ImageZoom
  src="/images/hero.jpg"
  alt="Hero banner"
  className="rounded-lg shadow-xl"
  imageClassName="object-cover"
/>
```

## Theming

ImageZoom uses the library's three-tier CSS custom property architecture:

1. **Core tokens** — raw design values (`--color-gray-500`, `--space-2`)
2. **Semantic tokens** — theme-aware mappings (`--color-bg-surface`, `--color-text-primary`)
3. **Component tokens** — component-specific overrides (`--image-zoom-*`)

Override any component token to customize the look:

```css
.my-wrapper .image-zoom {
  --image-zoom-bg: #0a0a1a;
  --image-zoom-border: #333;
  --image-zoom-placeholder-bg: #1a1a2e;
  --image-zoom-placeholder-text: #aaa;
}
```

Theme switching works automatically via the `data-theme` attribute on a parent element:

```html
<div data-theme="dark">
  <ImageZoom src="/photo.jpg" alt="Night sky" />
</div>
```

## Edge Cases

### Empty Source

When no `src` is provided, the component renders a placeholder state instead of an `<img>` element:

<Canvas of={ImageZoomStories.EmptySrc} />

### Image Load Failure

If the image fails to load, the `alt` text is displayed as visible fallback content inside the container, styled with the placeholder tokens.

### Decorative Images

Pass an empty `alt` string for decorative images. The container receives `role="img"` to maintain accessibility semantics.

```tsx
<ImageZoom src="/decorative-bg.jpg" alt="" />
```

## Accessibility

- The `alt` prop is applied directly to the `<img>` element
- Decorative images (`alt=""`) set `role="img"` on the container
- The container cursor is set to `zoom-in` to signal interactivity
- On image load failure, the alt text is rendered as visible fallback content
